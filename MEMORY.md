# Agent Town Rebuild Memory

Updated: 2026-05-30

## Discoveries

- The repository was not a Godot project at start of this continuation. It contained a React/Vite/Phaser frontend and FastAPI backend.
- Project docs conflicted: some files required Phaser, while the current user objective requires Godot.
- Godot was not installed on PATH or under the inspected top-level `D:\devtools` directories.
- Official/current release evidence indicated Godot 4.6.3 stable was the latest GitHub release on 2026-05-20, so the project-local tool copy uses 4.6.3 stable.
- The legacy `start.cmd` pointed at `D:\ai-town`, not the actual workspace `D:\Game_develop\ai-town`.
- FastAPI endpoint verification initially timed out because unbounded recursive knowledge-base scans are too expensive for game UI use.
- Claude review connector failed with a local `claude.exe` path recognition error, so no external Claude review evidence is available for this slice.

## Decisions

- Build a new `godot/` client instead of deleting the React/Phaser frontend immediately. This preserves usable assets and avoids breaking reference work.
- Use GDScript for fastest Godot iteration.
- Keep local system access read-only in the first slice. Real work execution will be added behind explicit safety gates.
- Ignore `tools/godot/` in git. It is a local runtime convenience, not source code for the open-source project.
- Bound all large local folder scans before exposing them to the game UI.
- The first playable work loop is intentionally read-only: accept quest -> visit building -> run safe scan -> complete quest. Write-capable actions still need explicit confirmation gates.
- A first room/workbench panel now exists. It is still an overlay, but it consumes `/api/workbench/rooms/{building_id}` payloads so it can evolve into dedicated room scenes without changing backend contracts.
- `tools/verify-smoke.ps1` is the current repeatable smoke test entry point.
- Quest completion now has a persistent reward loop: earned badges are saved in Godot's `user://agent_town_save.json` and shown in the Badge Case.
- Quests now have explicit step metadata and Godot tracks per-step progress. First supported chain: accept -> enter room -> review shelves -> safe scan -> badge.
- Room Mastery now adds persistent room-level progression. Entering buildings and completing safe workbench scans/drafts grant local XP, levels, and Room Mastery badges in `user://agent_town_save.json`; this is local player state only and does not call backend write endpoints or external services.
- NPC quest chains are now playable local progression. Godot stores `npc_chain_progress` in `user://agent_town_save.json`, marks stage checkboxes after successful safe room actions, completes chains, and awards `NPC Chains` badges without launching agents, running arbitrary commands, publishing GitHub state, exposing model secrets, or mutating source files.
